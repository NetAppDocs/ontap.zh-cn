= ONTAP Mediator 的新功能
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
每个版本都提供了对 ONTAP Mediator 的新增强功能。下面是新增功能。



[[enhancements]]
== 增强功能

有关SCST版本的信息，请参阅<<SCST支持列表>>。

[cols="25,75"]
|===


| ONTAP 调解器版本 | 增强功能 


 a| 
1.12.1
 a| 
* 支持RHEL：
+
** 兼容：10.1 和 9.7。
** 推荐：10.2、10.0、9.8、9.6、9.4 和 8.10。


* 支持 Rocky Linux 10.2、9.8 和 8.10。
* 支持使用 rhck 和 uek 内核的 Oracle Linux 10.1 和 9.8。
* 从 ONTAP Mediator 1.12.1 和 ONTAP 9.19.1 开始，配置了 HTTPS 的单个 ONTAP Mediator 实例可以同时监控以下配置：
+
** 如果安装在具有 8 核 CPU 和至少 16GB RAM 的主机上，则最多可安装 10 个四节点或 5 个八节点 MetroCluster IP 配置。请参阅 link:https://docs.netapp.com/us-en/ontap-metrocluster/install-ip/concept_mediator_requirements.html#mediator-multiple-mccip["使用 ONTAP Mediator 监控多个 MetroCluster IP 配置"^]。
** 如果安装在具有双核 CPU 的主机上，则为 1 个四节点 MetroCluster IP 配置和 1 个 SnapMirror 活动同步对等关系。请参阅 link:../snapmirror-active-sync/mediator-install-task.html#ontap-mediator["为SnapMirror主动同步配置ONTAP调解器"]。


* 包括用于修复损坏的 SCST 安装的新工具。安装 ONTAP Mediator 后，您可以在 `/usr/local/bin/mediator_repair_scst`找到该工具。如果升级后的操作系统需要默认 SCST 软件包中未提供的 SCST 修补程序，则可以在操作系统升级后使用该工具。




 a| 
1.12
 a| 
* 支持RHEL：
+
** 兼容：10.1 和 9.7。
** 推荐：10.2、10.0、9.8、9.6、9.4 和 8.10。


* 支持 Rocky Linux 10.2、9.8 和 8.10。
* 支持使用 rhck 和 uek 内核的 Oracle Linux 10.1 和 9.7。
* 支持配置使用 HTTPS 进行 MetroCluster IP 配置中通信的 ONTAP Mediator。HTTPS 通过使用 SSL/TLS 加密整个会话来提高安全性。您可以选择安装仅使用 HTTPS 协议的 ONTAP Mediator。如果使用此安装方法，则不会安装 SCST 包。link:index.html#https-iscsi-installation["详细了解 ONTAP Mediator 的 HTTPS 和 iSCSI 安装选项"]。




 a| 
1.11
 a| 
* 支持 RHEL 10.1、10.0、9.7、9.6、9.4 和 8.10。
* 支持 Rocky Linux 10.1、9.7 和 8.10。
* 支持使用 rhck 和 uek 内核的 Oracle Linux 10.0 和 9.6。
* 为MetroCluster IP 配置添加对 IPv6 的支持。
* 添加了对 fapolicyd 的支持。




 a| 
1.10
 a| 
* 支持 RHEL 10.0、9.6、9.4 和 8.10。
* 支持 Rocky Linux 10.0、9.6 和 8.10。
* 将 Python 基础版本从 Python 3.9 升级到 Python 3.12。




 a| 
1.9.1
 a| 
* 支持RHEL：
+
** 兼容：9.3、9.1、8.9、8.7、8.6、8.5 和 8.4。
** 推荐：9.5、9.4、9.2、9.0、8.10 和 8.8。


* 支持 Rocky Linux 9.5 和 8.10。
* 新增用于代码签名验证的证书。
* 增加了使用以下方式跳过代码签名检查的支持 `-skip-code-signature-check`旗帜。
* 安装程序在检测到过期代码签名证书时会显示警告。

1.9

* 支持RHEL：
+
** 兼容：9.3、9.1、8.9、8.7、8.6、8.5 和 8.4。
** 推荐：9.5、9.4、9.2、9.0、8.10 和 8.8。


* 支持 Rocky Linux 9.5 和 8.10。
* 对RHEL和洛基Linux的FIPS支持。
* 增加了性能增强功能、可实现更大的可扩展性。
* 改进了文件名，以简化PKI签名证书的设置。

1.8.
 a| 
* 支持RHEL：
+
** 兼容：8.7、8.6、8.5 和 8.4。

* 支持 Rocky Linux 9.4 和 8.10。

1.7.
 a| 
* 支持RHEL：
+
** 兼容：8.7、8.6、8.5 和 8.4。
** 推荐：9.3、9.2、9.1、9.0、8.9 和 8.8。


* 支持 Rocky Linux 9.3 和 8.9。
* 支持在自签名证书和第三方签名证书中使用SAN (使用者替代名称)数据。

1.6.
 a| 
* Python 3.9更新。
* 支持RHEL：
+
** 兼容：8.7、8.6、8.5 和 8.4。
** 推荐：9.2、9.1、9.0 和 8.8。


* 支持 Rocky Linux 9.2 和 8.8。
* 不再支持RHEL 7.x / CentOS所有版本。




 a| 
1.5.
 a| 
* 支持 RHEL 8.5、8.4、8.3、8.2、8.1、8.0、7.9、7.8、7.7 和 7.6。
* 支持 CentOS 7.9、7.8、7.7 和 7.6。
* 包括RHEL 7.x / CentOS 7.x的弃用警告
* 优化大规模SnapMirror活动同步系统的速度。
* 安装程序中添加了加密代码签名。




 a| 
1.4.
 a| 
* 支持 RHEL 8.5、8.4、8.3、8.2、8.1、8.0、7.9、7.8、7.7 和 7.6。
* 支持 CentOS 7.9、7.8、7.7 和 7.6。
* 增加了对基于UFEI的固件的安全启动(SB)的支持。




 a| 
1.3.
 a| 
* 支持 RHEL 8.3、8.2、8.1、8.0、7.9、7.8、7.7 和 7.6。
* 支持 CentOS 7.9、7.8、7.7 和 7.6。




 a| 
1.2.
 a| 
* 支持 RHEL 8.1、8.0、7.9、7.8、7.7 和 7.6。
* 支持 CentOS 7.9、7.8、7.7 和 7.6。
* 支持HTTPS邮箱。
* 与ONTAP 9.8+ MCC IP AUSO和SnapMirror活动同步的同步中的同步技术结合使用。

 a|

* 消除了对Perl的依赖。

* "是"表示建议安装ONTAP调解器的操作系统、该操作系统完全兼容并受支持。
* "否"表示操作系统和ONTAP 调解器不兼容。
* “兼容”意味着 Red Hat 不再支持这些 RHEL 版本，但仍可在这些版本上安装ONTAP Mediator。
* ONTAP Mediator 1.6 增加了对 Rocky Linux 9 和 8 的支持。
* ONTAP 调解器1.5是RHEL 7.x分支操作系统支持的最后一个版本。
* 由于CentOS 8已进行分支、因此已删除所有版本的CentOS 8。CentOS Stream被视为不适合生产目标操作系统。未计划提供支持。
