= SnapMirror 云备份到对象存储
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_ 是一种备份和恢复技术，专为希望将其数据保护工作流转换到云或本地对象存储解决方案的 ONTAP 用户而设计。放弃传统磁带备份架构的组织可以使用对象存储作为长期数据保留和存档的替代存储库。SnapMirror Cloud 提供 ONTAP 到对象存储的复制，作为增量永久备份策略的一部分。

SnapMirror Cloud 复制是获得许可的 ONTAP 功能。SnapMirror Cloud 是在 ONTAP 9.8 中引入的，作为 SnapMirror 复制技术系列的扩展。虽然 SnapMirror 经常用于 ONTAP 到 ONTAP 的备份，但 SnapMirror Cloud 使用相同的复制引擎将快照从 ONTAP 传输到符合 S3 标准的对象存储备份。

针对备份用例，SnapMirror Cloud 同时支持长期保留和归档工作流。与 SnapMirror 一样，初始 SnapMirror Cloud 备份会执行卷的基线传输。对于后续备份，SnapMirror Cloud 将生成源卷的快照，并将仅包含更改的数据块的快照传输到对象存储目标。

SnapMirror Cloud 关系可以在 ONTAP 系统与选定的本地和公共云对象存储目标之间配置，包括 Amazon S3、Google Cloud Storage 和 Microsoft Azure Blob Storage。其他本地对象存储目标包括 StorageGRID 和 ONTAP S3。

除了使用 ONTAP System Manager 管理 SnapMirror Cloud 配置外，还有几个编排选项可用于管理 SnapMirror Cloud 备份：

* 多个第三方备份合作伙伴提供对 SnapMirror 云复制的支持。参与的供应商可在 link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["NetApp 博客"^] 上找到。
* NetApp Backup and Recovery 是适用于ONTAP环境的NetApp原生解决方案
* 用于为数据保护工作流开发自定义软件或利用自动化工具的 API



NOTE: 由 System Manager 管理的 SnapMirror Cloud 复制仅支持将 StorageGrid 和 ONTAP S3 作为对象存储目标。不支持其他提供商。


NOTE: 使用 SnapMirror 云 API 通常仅限于备份合作伙伴。希望开发自己的基于 API 的解决方案的客户应联系其 NetApp 客户团队，了解许可条款的相关信息。
