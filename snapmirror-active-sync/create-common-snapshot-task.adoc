= 在 ONTAP 一致性组或 NAS SVM 之间创建通用快照
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
除了定期安排的快照操作之外，还可以在 SAN 配置中的主 SnapMirror 一致性组中的卷与辅助 SnapMirror 一致性组中的卷之间，或在 NAS 配置的 SVM 级别上手动创建公共 link:../concepts/snapshot-copies-concept.html["Snapshot"]。

.关于此任务
自动故障转移策略的计划快照创建间隔为 12 小时，自动故障转移双工策略的计划快照创建间隔为 3 小时。

.开始之前
* SnapMirror 组关系必须处于同步状态。


.步骤
您可以使用 System Manager 或 ONTAP CLI 创建通用快照。

[role="tabbed-block"]
====
.System Manager
--
. 在目标集群中，导航到 *Protection > Replication*。
. 找到关系源，选择 image:icon_kabob.gif["菜单选项图标"] 并选择 *Update*。
. 选择*根据策略*或*选择快照*以选择现有快照。
. 选择 *Update*。


--
.命令行界面
--
. 从目标站点创建一个通用快照：
+
`snapmirror update -destination-path <destination path>`

+
SAN 配置示例：

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
NAS 配置示例：

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. 监控更新进度：
+
`destination::>snapmirror show -fields newest-snapshot`
