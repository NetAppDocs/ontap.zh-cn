= 使用ONTAP系统配置FC或FC-NVMe网络结构
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
对于 FC 和 FC/NVMe，存储解决方案和主机之间的网络拓扑称为结构。要创建冗余以防止数据访问丢失，建议您在多结构配置中使用两个或多个交换机设置具有 HA 对的 SAN。在主机和存储之间仅使用一个交换机的单结构配置不是完全冗余的。

从 ONTAP 9.19.1 开始，您还可以直接将 FC 或 FC-NVMe SAN 主机连接到 HA 对，而无需使用交换机。在简单性至关重要的远程站点和混合环境中，直接连接配置尤其有价值。

级联、部分网状、全网状、核心边缘和导向器结构都是将 FC 交换机连接到结构的行业标准方法，并且都受支持。除嵌入式刀片式交换机外，不支持使用异构 FC 交换机结构。特定的例外情况在 link:https://imt.netapp.com/matrix/["互操作性表工具"^] 上列出。结构可以由一个或多个交换机组成，存储控制器可以连接到多个交换机。

使用不同操作系统（如 Windows、Linux 或 UNIX）的多个主机可以同时访问存储控制器。主机要求安装和配置受支持的多路径解决方案。可以在互操作性矩阵工具上验证受支持的操作系统和多路径解决方案。



== 多结构FC和FC-NVMe配置

建议您使用 HA 对和至少两台交换机配置 FC 和 FC-NVMe SAN 主机。[EH5.1][FD5.2][EH5.3]这在结构和存储系统层提供冗余，以支持容错和无中断操作。

在多结构 HA 对配置中，两个或多个交换机将 HA 对连接到一个或多个主机。每台交换机都会创建一个单独的结构，该结构由主机与通过交换机的 HA 对之间的路径组成。添加交换机会创建额外的结构。多结构配置可以包括两个或多个交换机及其相应的结构。为简单起见，下图仅显示两个交换机，它们代表两个独立的结构。

例如、插图中的FC目标端口号(0c、0d、1a、1b)。实际端口号因存储节点的型号以及是否使用扩展适配器而异。

image:scrn_en_drw_fc-32xx-multi-HA.png["多网络结构 HA 对示意图"]



== 单网络结构FC和FC-NVMe配置

在单结构 HA 对配置中，一台交换机将 HA 对连接到一台或多台主机。单个交换机创建由主机和 HA 对之间的所有路径组成的单个结构。由于没有冗余，交换机故障可能会导致服务中断。

例如、图中的FC目标端口号(0a、0c)。实际端口号因存储节点的型号以及是否使用扩展适配器而异。

支持FC配置的所有平台均支持单网络结构HA对配置。

image:scrn_en_drw_fc-62xx-single-HA.png["单交换机网络结构 HA 对示意图"]



== 直连式 FC 和 FC-NVMe 配置

从 ONTAP 9.19.1 开始，直接连接 FC 和 FC-NVMe 配置可用于 AFF、ASA、ASA r2 和 FAS 系统。在直接连接配置中，一个或多个主机直接连接到 HA 对，而无需交换机。在简单性至关重要的远程站点和混合环境中，这种配置尤其有价值。

image:scrn_en_drw_fc-302020-direct-sing-on.png["直连配置"]
