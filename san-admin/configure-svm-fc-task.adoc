= 为 FC 配置 SVM
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
要为 FC 配置 Storage Virtual Machine （ SVM ），必须为 SVM 创建 LIF 并将 FC 协议分配给这些 LIF 。

.关于此任务
对于使用 FC 协议提供数据的每个 SVM ， NetApp 至少支持每个节点一个 FC LIF 。  每个节点必须使用两个 LIF ，并使用两个网络结构，每个节点连接一个 LIF 。这样可以在节点层和网络结构上实现冗余。

.开始之前
* 您必须是集群管理员才能执行此任务。
* 您必须拥有FC许可证link:../system-admin/manage-licenses-concept.html#licenses-included-with-ontap-one["随ONTAP One提供"]()，并且必须启用该许可证。如果未启用FC许可证，LIF和SVM将显示为联机，但运行状态将为 `down`。要使 LIF 和 SVM 正常运行，必须启用 FC 服务。要托管启动程序，必须对 SVM 中的所有 FC LIF 使用单个启动程序分区。


.步骤
[role="tabbed-block"]
====
.System Manager
--
使用ONTAP System Manager (9.7及更高版本)为iSCSI配置Storage VM。

[cols="2"]
|===
| 在新的Storage VM上配置FC | 在现有Storage VM上配置FC 


 a| 
. 在 System Manager 中，选择 *Storage > Storage VMs*，然后选择 *Add*。
. 输入Storage VM的名称。
. 为*访问协议*选择* Fc*。
. 选择 *Enable FC*。
+
FC 端口将自动分配。

. 选择 * 保存 * 。

 a| 
. 在 System Manager 中，选择 * 存储 > 存储 VM* 。
. 选择要配置的存储 VM。
. 在 *Settings* 选项卡上选择，然后选择 FC 协议旁边的 image:icon_gear.gif["操作图标"]。
. 选择 *Enable FC* 并输入网络接口的 IP 地址和子网掩码。+ FC 端口将自动分配。
. 选择 * 保存 * 。


|===
--
.命令行界面
--
. 在 SVM 上启用 FC 服务：
+
[source, cli]
----
vserver fcp create -vserver <vserver_name> -status-admin up
----
. 在为 FC 提供服务的每个节点上为 SVM 创建两个 LIF ：
+
** 对于ONTAP 9.6及更高版本：
+
[source, cli]
----
network interface create -vserver <vserver_name> -lif <lif_name> -data-protocol fcp -service-policy default-data-fcp -home-node <node_name> -home-port <port_name> -address <ip_address> -netmask <netmask> -status-admin up
----
** 对于ONTAP 9.5及更早版本：
+
[source, cli]
----
network interface create -vserver <vserver_name>_ -lif <lif_name> -role data -data-protocol fcp -home-node <node_name> -home-port <port_name>
----


. 确认已创建LIFs且其运行状态为 `online`：
+
[source, cli]
----
network interface show -vserver <vserver_name> <lif_name>
----
+
有关的详细信息 `network interface show`，请参见link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP 命令参考"^]。
