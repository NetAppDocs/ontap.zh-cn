== 安全性

[cols="30%,70%"]
|===
| 更新 | Description 


 a| 
xref:../anti-ransomware/use-cases-restrictions-concept.html#supported-configurations[FlexGroup卷支持 ARP/AI]
 a| 
NetApp FlexGroup卷支持 ARP/AI，提供基于机器学习的高级勒索软件检测和保护。在ONTAP 9.17.1 及更早版本中， FlexGroup卷仅支持第一代 ARP。从ONTAP 9.18.1 开始， FlexGroup卷在本地ONTAP平台（AFF和FAS）和虚拟ONTAP部署（包括Cloud Volumes ONTAP和ONTAP Select ）上与FlexVol卷具有同等功能。因此，ARP/AI 成为FlexVol和FlexGroup卷的默认 ARP 技术。



 a| 
xref:../anti-ransomware/index.html#enablement-options-for-arp[在受支持的系统上，新卷的 ARP/AI 默认处于启用状态]
 a| 
当您创建新集群或将集群升级到 9.18.1 时，在 12 小时的宽限期后，ARP/AI 会在 AFF A 系列、AFF C 系列、ASA 和 https://docs.netapp.com/us-en/asa-r2/secure-data/enable-anti-ransomware-protection.html["ASA r2"]系统的所有新卷上默认自动启用。在此宽限期内，您可以选择退出默认启用。

|===


== 存储资源管理增强功能

[cols="30%,70%"]
|===
| 更新 | Description 


 a| 
xref:../performance-admin/adaptive-qos-policy-groups-task.html[支持在SVM和卷上设置两级嵌套的QoS策略组]
 a| 
您可以同时为 SVM 和 SVM 下的卷分配 QoS 策略。



 a| 
xref:../flexcache/create-volume-task.html[在 SVM-DR 关系中创建源卷的 SVM FlexCache卷]
 a| 
您可以创建属于 SVM-DR 关系的源卷的 SVM FlexCache卷。



 a| 
xref:../volumes/commands-display-space-usage-reference.html[用于显示空间使用情况信息的命令]
 a| 
在 ONTAP 9.18.1 及更高版本中， `storage aggregate show-space` 命令更改逻辑参考容量和逻辑未参考容量的报告方式。Logical Referenced Capacity 报告所有对象中引用的块和碎片化对象中未引用的块。Logical Unreferenced Capacity 仅报告已超过满度阈值且有资格进行对象删除和碎片整理的对象中的未引用块。

例如，当您对 ONTAP S3 和 StorageGRID 使用默认聚合满度阈值 40% 时，在将块报告为未引用容量之前，必须取消引用对象中 60% 的块。

在早于 ONTAP 9.18.1 的版本中，逻辑引用容量报告所有对象（完整和碎片对象）中的引用块。逻辑未引用容量报告所有对象中的未引用块。
