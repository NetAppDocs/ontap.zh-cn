= 使用 ONTAP SnapMirror Cloud 将数据备份到云
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_ 提供到对象存储目标的 ONTAP 复制。这使您可以用对象存储取代旧的磁带备份架构，以实现长期数据保留和存档。您可以使用 Amazon S3、Alibaba Cloud、Azure Blob、Google Cloud Storage、IBM Cloud Object Storage、NetApp StorageGRID 或 ONTAP S3 等提供商作为云对象存储。


NOTE: 由 System Manager 管理的 SnapMirror Cloud 复制仅支持将 StorageGrid 和 ONTAP S3 作为对象存储目标。不支持其他提供商。


NOTE: 支持将 SnapMirror 云复制到其他对象存储提供商以及其他备份和恢复产品的情况因供应商而异。请咨询您的第三方供应商，了解其支持政策。


NOTE: 使用 SnapMirror 云 API 通常仅限于备份合作伙伴。希望开发自己的基于 API 的解决方案的客户应联系其 NetApp 客户团队，了解有关许可条款的信息。

从 ONTAP 9.19.1 开始：

* SnapMirror Cloud 将每个集群支持的 S3 对象存储桶数量从 6 个增加到 100 个。此限制适用于在 ONTAP 集群上的所有 SnapMirror Cloud 对象存储端点上配置的对象存储总数。使用此功能时，请注意以下事项：
+
** 当每个集群使用超过 6 个 bucket 时，建议对支持该功能的服务商使用虚拟主机风格的 URL 配置，包括 AWS S3、Alibaba Cloud、StorageGrid、Google Cloud Storage、IBM Cloud Object Storage、ONTAP S3 以及其他兼容 S3 的服务商。推荐使用此配置以实现可扩展的对象存储寻址，并避免底层限制。
** 默认情况下，按需还原处于启用状态。
** 根据系统中有多少并发恢复处于活动状态，按需恢复期间的读取延迟可能会更高。




从 ONTAP 9.18.1 开始：

* SnapMirror Cloud 支持使用现有的 link:https://docs.netapp.com/us-en/ontap-automation/get-started/access_rest_api.html["ONTAP REST API"^] 对 MetroCluster 配置上的 FlexGroup 卷进行备份和恢复操作。此功能使您能够为 MetroCluster 配置上的 FlexGroup 卷创建 SnapMirror Cloud 关系，这些关系在切换和切回后仍由伙伴站点维护。


在使用 SnapMirror Cloud 功能之前，您应该从 NetApp 支持站点申请 SnapMirror Cloud API 许可证密钥：link:https://mysupport.netapp.com/site/tools/snapmirror-cloud-api-key["请求 SnapMirror Cloud API 许可证密钥"^]。

按照说明，您应提供业务机会的简单描述，并通过向提供的电子邮件地址发送电子邮件来请求 API 密钥。您应会在 24 小时内收到电子邮件回复，其中包含有关如何获取 API 密钥的进一步说明。


NOTE: 您可能需要 SnapMirror Cloud 许可证密钥和 API 密钥。虽然 ONTAP One 包含 SnapMirror Cloud 许可证本身，但需要许可证密钥才能启用 SnapMirror Cloud 功能。可能需要单独生成和安装许可证密钥。



[[add-a-cloud-object-store]]
== 添加云对象存储

在配置 SnapMirror 云备份之前，需要添加 StorageGRID 或 ONTAP S3 云对象存储。

.步骤
. 单击 * 保护 > 概述 > 云对象存储 * 。
. 单击 image:icon_add.gif["添加图标"]。




[[back-up-using-the-default-policy]]
== 使用默认策略进行备份

您可以使用默认的云保护策略 DailyBackup 快速为现有卷配置 SnapMirror 云备份。

.步骤
. 单击 * 保护 > 概述 * ，然后选择 * 将卷备份到云 * 。
. 如果这是您第一次备份到云，请在许可证字段中输入您的 SnapMirror Cloud API 许可证密钥，如图所示。
. 单击 * 身份验证并继续 * 。
. 选择源卷。
. 选择云对象存储。
. 单击 * 保存 * 。




[[create-a-custom-cloud-backup-policy]]
== 创建自定义云备份策略

如果您不想为 SnapMirror Cloud 备份使用默认的 DailyBackup 云策略，可以创建自己的策略。

.步骤
. 单击 * 保护 > 概述 > 本地策略设置 * ，然后选择 * 保护策略 * 。
. 单击 * 添加 * 并输入新策略详细信息。
. 在 * 策略类型 * 部分中，选择 * 备份到云 * 以指示您正在创建云策略。
. 单击 * 保存 * 。




[[create-a-backup-from-the-volumes-page]]
== 从*卷*页面创建备份

如果要一次为多个卷选择并创建云备份、或者要使用自定义保护策略、可以使用System Manager *卷*页面。

.步骤
. 单击 * 存储 > 卷 * 。
. 选择要备份到云的卷，然后单击 * 保护 * 。
. 在 * 保护卷 * 窗口中，单击 * 更多选项 * 。
. 选择一个策略。
+
您可以选择默认策略 DailyBackup 或您创建的自定义云策略。

. 选择云对象存储。
. 单击 * 保存 * 。




[[restore-from-the-cloud]]
== 从云还原

您可以使用 System Manager 将已备份的数据从云存储还原到源集群上的其他卷。


NOTE: 如果使用的是 ONTAP 9.16.1 或更高版本，并且要执行 SnapMirror Cloud 单个文件还原到 FlexGroup 卷，则应仅将文件还原到 FlexGroup 卷中的新目录，并且必须在目标 FlexGroup 卷上将粒度数据设置为 `advanced`。有关设置 `-granular-data advanced`选项的详细信息，请参见link:../flexgroup/enable-adv-capacity-flexgroup-task.html["通过重新分布文件数据来平衡ONTAP FlexGroup卷"]。

.步骤
. 从SnapMirror到云关系的源集群中、单击*存储>卷*。
. 选择要还原的卷。
. 选择 * 备份到云 * 选项卡。
. 单击 image:icon_kabob.gif["菜单选项图标"] 要还原的源卷旁边的以显示菜单，然后选择*Restore*。
. 在 * 源 * 下，选择一个 Storage VM ，然后输入要将数据还原到的卷的名称。
. 在*目标*下、选择要还原的快照。
. 单击 * 保存 * 。




[[delete-a-snapmirror-cloud-relationship]]
== 删除 SnapMirror Cloud 关系

您可以使用 System Manager 删除云关系。

.步骤
. 单击 * 存储 > 卷 * ，然后选择要删除的卷。
. 单击 image:icon_kabob.gif["菜单选项图标"] 源卷旁边的并选择*Delete*。
. 如果要删除云对象存储端点，请选择 * 删除云对象存储端点（可选） * 。
. 单击 * 删除 * 。




[[remove-a-cloud-object-store]]
== 删除云对象存储

如果云对象存储不属于云备份关系，您可以使用 System Manager 将其删除。如果云对象存储属于云备份关系，则无法将其删除。

.步骤
. 单击 * 保护 > 概述 > 云对象存储 * 。
. 选择要删除的对象存储，单击 image:icon_kabob.gif["菜单选项图标"] 并选择*Delete*。
