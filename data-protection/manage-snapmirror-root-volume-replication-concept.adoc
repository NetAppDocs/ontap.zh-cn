= 了解ONTAP SnapMirror根卷复制
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NAS 环境中的每个 SVM 都有一个唯一的命名空间。SVM 根卷是此命名空间层次结构的入口点。对于由两个或多个 HA 对组成的集群，应考虑 SVM 根卷的负载共享镜像 (LSM)，以确保在 HA 对的两个节点都出现故障时，客户端仍然可以访问命名空间。负载共享镜像不适用于由单个 HA 对组成的集群，也不适用于 MetroCluster 环境。

* 从 ONTAP 9.1 开始，数据卷的负载共享镜像已被弃用，仅支持 SVM 根卷。
* SnapMirror 负载共享镜像只能支持 NAS (CIFS/NFSv3) 客户端。负载共享镜像不支持 NFSv4 客户端或 SAN 客户端协议连接 (FC、FCoE 或 iSCSI)。但是，NFSv4 和负载共享镜像可以在同一环境中共存。
