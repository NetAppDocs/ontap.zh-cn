= 了解ONTAP网络上的LIF故障转移
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
LIF 故障转移是指在 LIF 的当前端口发生链路故障时， LIF 自动迁移到其他网络端口。这是为 SVM 连接提供高可用性的一个关键组件。配置 LIF 故障转移包括创建故障转移组，修改 LIF 以使用故障转移组以及指定故障转移策略。

SAN 协议 LIF 不支持使用故障转移组的 LIF 故障转移。了解link:../san-admin/multipath-automatic-lif-failover-support.html["SAN 多路径和自动 LIF 故障转移支持"]。

故障转移组包含来自 ONTAP 集群中一个或多个节点的一组网络端口（物理端口、VLAN 和接口组）。故障转移组中存在的网络端口定义了 LIF 可用的故障转移目标。故障转移组可以为其分配集群管理 LIF、节点管理 LIF、集群间 LIF 和 NAS 数据 LIF。


IMPORTANT: 如果在配置 LIF 时没有有效的故障转移目标，则在 LIF 尝试进行故障转移时会发生中断。您可以使用 `network interface show -failover`命令验证故障转移配置。有关的详细信息 `network interface show`，请参见link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP 命令参考"^]。

创建广播域时，系统会自动创建一个同名的故障转移组，该故障转移组包含相同的网络端口。此故障转移组由系统自动管理，这意味着在从广播域添加或删除端口时，这些端口会自动添加或从此故障转移组中删除。对于不想管理自己的故障转移组的管理员来说，这是一种效率。

从 ONTAP 9.19.1 开始，ONTAP 系统定义的故障转移组策略动态扩展 NAS 数据 LIF 的故障转移目标，可能包括集群中任何可用节点上广播域中的任何可用端口。您可以将系统定义的策略应用于任何故障转移组。在 ONTAP 9.19.1 之前，使用此策略的 LIF 只能故障转移到集群中的另一个节点；如果该节点超出法定人数，则 LIF 不会故障转移，即使有其他节点可用。
