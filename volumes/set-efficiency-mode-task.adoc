= 在创建卷期间设置存储效率模式
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
从 ONTAP 9.10.1 开始，您可以在创建新 AFF 卷时设置存储效率模式。

.关于此任务
您可以使用以下参数控制新AFF卷的存储效率模式 `-storage-efficiency-mode`。您可以选择以下两种选项来设置存储效率模式： `default` 或者 `efficient`。您选择的存储效率模式取决于您想要更高的性能还是更高的存储效率。参数 `-storage-efficiency-mode` 在非AFF卷或数据保护卷上不受支持。

创建新的AFF卷时，如果启用存储效率模式，则默认设置为性能模式。

link:enable-temperature-sensitive-efficiency-concept.html["详细了解对温度敏感的存储效率和存储效率模式"](英文)

.步骤
. 创建新卷并设置效率模式：
+
[source, cli]
----
volume create -vserver <vserver name> -volume <volume name> -aggregate <aggregate name> -size <volume size> -storage-efficiency-mode <efficient|default>
----
+
放 `-storage-efficiency-mode` 到 `efficient` 为了提高效率模式或 `default` 用于性能模式。

+
在以下示例中，aff_vol1 是以效率模式创建的。



[listing]
----
volume create -vserver vs1 -volume aff_vol1 -aggregate aff_aggr1 -storage-efficiency-mode efficient -size 10g
----
